5  Create your first Quarto document

5.1 Installation

To use Quarto with R, you should install the rmarkdown R package:

install.packages(“rmarkdown”)

Installation of the rmarkdown package will also install the knitr package so you will have everything required to render documents containing R code.

Quarto will select a version of R by looking on the system PATH. In addition, on Windows when R is not found on the PATH, the registry will be scanned for the current R version. You can override the version of R used by Quarto by setting the QUARTO_R environment variable.

5.2 Create a new Quarto document

In RStudio, you can open a new Quarto document by selecting File > New File > Quarto Document…

When you create a new Quarto document, RStudio tries to be helpful by allowing you to select a template which explains the different section of an R Markdown script. R Studio will enable you select options to pick from to generate a template Quarto document to start from.

The title and the author names are not important. If the output document type you want is not one of these, do not worry - you can just pick any one and change it later.

Let us select HTML to create an html document.

Click on create to open up a new Quarto (.Qmd) document.

5.3 Visual Editor

The RStudio Visual Editor is quite new and has features that improve your writing experience. Working in the Visual Editor feels a bit like working in a Google Doc.

5.4 Quarto document structure

An R Markdown document can be edited in RStudio.

There are three basic components to an Quarto document, similar to the components of a R Markdown document:

  • a YAML header
  • Markdown text
  • R code chunks

The header contain the information intended to produce an html output.

5.5 Quarto render

When you click the Render button a document will be generated that includes both content and the output of embedded code.

5.6 Code options

You can embed code like this:

1 + 1
[1] 2

You can add options to executable code like this

[1] 4

The echo: false option disables the printing of code (only output is displayed).
